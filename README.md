# Webサイト：My Portfolio

![My Portfolio サムネイル](./img/docs/thumbnail-my-portfolio.jpg)

## 概要

Webコーダーとしてのスキルと制作実績を整理し、第三者に分かりやすく伝えることを目的に制作したポートフォリオサイトです。  

作品への導線設計と回遊性を重視し、KVスライダーや作品詳細ページ間のページネーションを実装しました。  
また、ダーク / ライトのテーマ切り替えやページ遷移演出など、サイト全体の体験を意識して設計・実装しています。

## 設計

本サイトでは「作品へのアクセス性」を最重要とし、トップページから最小限の操作で作品詳細へ到達できる構成としました。

Worksセクションに加え、KVスライダーからも各作品へ遷移できる導線を設け、複数の入口を用意しています。

また、作品詳細ページにはページネーションを設置し、一覧に戻らずに作品間を回遊できる設計としました。前後の作品名を表示することで、次の遷移先が直感的に分かるようにしています。

---

## 工夫した点

- **作品へ最短で辿り着ける導線設計**  
  WorksだけでなくKVスライダーからも各作品へアクセスできる構成とし、閲覧者が迷わず回遊できるように設計

- **作品詳細を回遊できるページネーション**  
  一覧に戻らずに作品間を移動できるUIとし、前後の作品名を表示することで次の行動を明確化

- **ページを跨いでも崩れないスムーススクロール**  
  固定ヘッダー分のオフセットを考慮し、別ページからのハッシュ遷移でも同じ位置にスクロールされるよう調整

- **テーマ切り替えの体験と一貫性**  
  localStorageを用いてテーマ状態を保持し、再訪時もユーザーの選択を維持。ロゴやアイコンも連動して切り替え

## 技術選定の理由

アニメーションには表現力と制御性を重視してGSAPを採用しました。  
テキストスライダーにはSplideを使用し、シンプルかつ拡張性のある実装としています。

また、JavaScriptは機能ごとにコンポーネント化し、可読性と保守性を意識した構成としました。

## 学び

本制作を通して、作品を「見てもらう」ためには、単なるデザインやコーディングだけでなく、情報設計や導線、回遊性といった体験設計が重要であると実感しました。

また、テーマ切り替えやページ遷移などの全体に影響する機能では、状態管理（localStorageなど）とコンポーネント設計が品質に大きく関わることを学びました。

アニメーションについても、視覚的な演出だけでなく、操作の分かりやすさや体験の滑らかさを補助する役割として設計する重要性を理解しました。
